Loops JavaScript

O que é

Um Loop em javascript é um comando que se repete diversas automaticamente durante o código.

Loops tem 3 principais partes.

  • Loop Variable
  • Loop Condition
  • Increment Step

Para fazer um loop parar, temos que criar uma variável que controla e fazer ele parar na condição dentro dos parenteses.

Cada vez que um loop roda, é chamado de iteração

Loops criam um escopo, qualquer variável criada entre as chaves fica entre as chaves.

Tipos de Loops

WHILE

While é uma estrutura de repetição em que o teste lógico se faz no Início, antes de executar o bloco


Fonte: Curso em Vídeo Javascript - Gustavo Guanabara 

    while(condição){
        alert('Repetir este bloco')
    }

    let n = 10
    while(n > 0){
        console.log(n)
        n--
    }

DO-WHILE

DO-WHILE é uma estrutura de repetição que faz o teste lógico no final, após executar o bloco


Fonte: Curso em Vídeo Javascript - Gustavo Guanabara 

    do{
        alert('Repetir este bloco')
    }while(Condição)

    do{
        
    }

FOR

FOR é uma estrutura de repetição que possue dentro da sua estrutura uma variável que define quantas vezes será executado o bloco

Possui como argumentos 3 partes. A primeira recebe a Loop Variable, a segunda a Loop Condition, e a terceira a Increment Step


    for(loop variable; loop condition; increment step){

    }

Fonte: Curso em Vídeo Javascript - Gustavo Guanabara 

    for(declarar variável; Condição para a variável; O que fazer com a variável){

    }

    for(let n = 1; n < 10; n++){
        console.log(n)
    }

Quando usa cada tipo de Loop

Recomenda-se que quando vamos fazer algum loop padrão (Que possua as 3 principais partes), usemos For, pois fica mais organizado já que a Loop Variable, Loop Condition, e Step Increment estão juntas no mesmo lugar. Quando não há as 3 principais partes, usamos While

ex: 

    let randomNumber = 0

    while (randoNumber < 0.5){
        randomNumber = Math.random()
    }

Esse Loop não possui Step Increment.

FOR IN

For in é um loop especial para arrays, que faz com que repita o loop de ac acordo à quantidade de elementos de um array.

Estrutura: 

    for (__ in __){

    }

O primeiro parâmetro recebe a declaração de uma variável, segundo recebe o nome do array.

ex: 

    let array = [1, 8, 3, 8]
    for(i in array){
        console.log(array[i])
    }

MAP / SET

Array, Map e Set são tipos especiais de objetos JavaScript. Cada um tem características diferentes, que permite com que sejam usados para coisas diferentes.

MAP

MAP é uma estrutura de dados que existe em outras linguagens de programação, e no ES6 foi implementado no JavaScript

Map é uma coleção de pares. Cada par de um Map é um Array

Declarar MAP

Para declarar um Map, usa-se 'new Map()'

const myMap = new Map()

Podemos declarar um Map com valores usnaod forma de arrays.


    const myMap = new Map([
    [1, 'Lucas'],
    [2, 'João'],
    [3, 'Roberto']
    ])

MAP Loop

A estrutura de repetição for(__ of map_name){ } é como o for in para Maps.


    const myMap = new Map([
    [1, 'Lucas'],
    [2, 'João'],
    [3, 'Roberto']
    ])

    for(i of myMap){
        console.log(i)
    }

Como cada par do map é um array, podemos chamar as chaves e os valores através da notação de array


    const myMap = new Map([
    [1, 'Lucas'],
    [2, 'João'],
    [3, 'Roberto']
    ])

    for(i of myMap){
        console.log(i[0]) | Retorna só as chaves
        console.log(i[1]) | Retorna só os valores
    }

Características de um SET

Um Set não permite valores duplicados. Se um valor for repetido ele apenas ignora o valor.

Um Set pode conter arrays e objetos.

Os elementos de um Set, diferente de um array, não possuem index, então não tem como acessá-los. Set não foi criado para substituir arrays, ele foi criado para dar suporte à coisas que um array não consegue fazer, como remover duplicatas.

Funções

Como Funcionam:

É um bloco de código que é executado somente quando nós chamarmos a função.

Para definir uma função usa-se a palavra function, seguida do nome da função, abre-fecha parenteses, abre-fecha chaves function andar(){}.

O que está dentro dos parênteses se chama parâmetro, o que está dentro das chaves, bloco, é o que será executado quando a função for chamada.

Para chamar uma função, basta escrever o nome dela seguido de abre-fecha parenteses.

ex: 
    function andar(){
        window.alert('andou')
    }
    andar()

Qualquer variável criada dentro do escopo de uma função só existe dentro dele.

return:

Para tirar um valor de dentro de uma função usa-se return. Quando uma função é chamada, ela ecexuta o que tem dentro do bloco, e o valor de retorno o valor colocado no return. O comando return finaliza a função. Nada escrito após o return será lido. 


    function calcular(){
        let n = 2+2
        return n
    }

    Função calcular() vale n

Parâmetro:

O parâmetro é uma variável que é criada dentro da função e é passado no momento que se chama a função

ex:

        return n
    }
    console.log(elevar_ao_quadrado(5)) | Resutado = 25

Quando se coloca dentro dos parenteses que um parâmetro recebe algo, isso significa que caso o valor não seja definido, será considerado este passado dentro dos parâmetros 


    function somar(a=0, b=0){

Funções Avançadas

Funções são valores como quaisquer outros, isso significa que podemos armazenálas em variáveis

ex:

    }
    console.log(typeof function1) | Restulado: function
    function1() | Chamando a função

Como a função está dentro de uma variável, não é necessário que ela tenha um nome, pois ja está associada à variável. Isso é chamado de uma função anônima.

O jeito de declarar função function name(){} é um atalho do var name = function (){}


    const function1 = function (){
        console.log('Helo')
    }
    function1() | Chamando a função

Essa forma de abreviar funções tem 2 vantagens.

  1. Mais facil de ler
  2. Hoisting

Hoisting significa que uma função pode ser chamada antes mesmo de ser criada.

Funções são valores, então podemos colocá-las em objetos

Funções podem ser chamadas dentro de outras fuções

ex: 

    function run(param){

        param()
    }  
    
    run(
        function(){
            console.log('Running')
        }
        )

Aqui está sendo passado como parâemtro da função run uma função anônima que escreve Running no console

A função que é passada como parâmetro de outra função é chamada Callback

Código Assíncrono

Código Síncrono -> O computador espera terminar de ler o primeiro código para ler o próximo

Código Assíncrono -> O computador não vai esperar terminar de ler um código para começar a ler o próximo código

A partir do momento que esse comando é lido ele inicia o contador e segue lendo o resto do código. Quando o contador chegar no tempo escolhido ele ativa essa função. Isso se chama Asynchronous Code (código assíncrono).

COMANDO Função Exemplo

        console.log('Olá')
    }, 5000)
setInterval(__, __) Repete a função a cada determinado tempo. Par1 recebe a função, par2 recebe o tempo 
setInterval(function(){
        console.log('Olá')
    }, 2000)

Arrow Functions

Arrow Function é um shorcut para funções anônimas. Arrow Functions tem algumas vantagens sobre funções anônimas normais

Função Comum: 

    const regularFunction = function(param1, param2){
        console.log('Regular Function')
        return 5
    }
Arrow Function: 

    const arrowFunction = (param1, param2) => {
        console.log('Arrow Function')
        return 5
    }

Arrow Functions tem alguns shortcuts que funções normais não tem.

Quando uma Arrow Function tem apenas um parâmetro, os parenteses são opcionais

ex: 

    const oneParamFunction = (param) => {
        console.log('Arrow Function')
        return 5
    }
    oneParamFunction(10)



    const oneParamFunction = param => {
        console.log('Arrow Function')
        return 5
    }
    oneParamFunction(10)

Quando uma Arrow Function tem apenas uma linha, podemos colocar tudo na mesma linha da seta, apagar o return e as chaves

ex: 

    const oneLineFunction = () => {
        return 2 + 3
    }
    oneLineFunction()



    const oneLineFunction = () => 2 + 3
    oneLineFunction()

Quando um método é criado dentro de um objeto, existe uma maneira mais simples de declarar

ex: 

    const person = {
        name: 'Robson',
        speak: function(){
            console.log(this.name + ' spoke')
        }

Como Fuciona:

O Built-in object localStorage é usado para salvar dados permanentemente. Quando fechamos uma aba de um site, ele automaticamente zera todos os dados gerados no site. Quando salvamos estes dados no localStorage, mesmo após fechar e abrir a página os dados continuam.a

Comandos

COMANDO Função Característica

Podemos remover propriedades de um objeto e salvá-las em variáveis, por exemplo, isso se chama 'Destructuring':

RegEx - Regular Expressions

O que é RegEx

RegEx é uma notação para estabelecer padrões te textos ou números. Tem 3 principais aplicações.

  • Efetuar Buscas de informações com um formato específico
  • Efetuar Validações de texto. (Verifica se um texto atende a um padrão estabelecido)
  • Fazer Substituições

Características

Toda Expressão Regular tem que ficar entre barras / / Isso faz a linguagem entender que isso é uma RegEx

Existem funções em JavaScript para se interagir com RegEx. texto.match(RegEx) e RegEx.exec(texto)

RegEx pode ser utilizado em qualquer sequência de caracteres, seja o parágrafo de um livro (DOC), um texto estruturado como XML, JSON ou até em um código como por exemplo CSS

Comandos JavaScript de interação com RegEx

COMANDO Função Exemplo
__.match(__) Usa o texto e testa na expressão regular
O primeiro valor recebe o texto. O segundo a RegEx.
Caso contenha essa expressão, retorna um array com as palavras, caso contrário retorna null		 let text = 'Este é um texto'
let regex = /texto/g
let results = text.match(regex)
__.exec(__) Usa a expressão regular para testar o texto.
O primeiro valor recebe a RegEx, o segundo o texto,		 let text = 'Este é um texto'
let regex = /texto/g
let results = regex.exec(text)

Notação RegEx

Flags

Flag é a letra que vai após o final da expressão regular, ela faz com que tenha algumas características.

ex: /texto/g

Flags Importantes:

FLAG Função
Por padrão busca apenas a primeira palavra com o padrão
g General -> Busca todas as palavras com este padrão
i Insensitive -> Ignora se a letra for maiúscula ou minúscula
m Multiline -> Em textos de várias linhas, faz com que ^ deixe de valer só no início do texto e comece a valer para todas as linhas

Formas de notação

[] -> Busca pelo caractere de dentro indidualmente.

{} -> Recebe Número de ocorrêcias do padrão.

() -> Caracteriza um grupo. Para chamá-lo usase $ com o número do grupo.

^ ->caracteriza o início da linha.

$ -> caracteriza o fim da linha.

. -> faz com que aceite qualquer tipo de caractere.

| -> serve como OU.

? -> O padrão pode ocorrer zero ou uma vez.

* -> O padrão pode ocorrer 0 ou várias vezes.

+ -> O padrão pode ocorror 1 ou mais vezes.

Exemplos:

Email: abcde12.34@gmail.com.br
RegEx Email:/^[A-Za-z._%+-]+@[a-zA-z0-9.-]+\.[a-zA-z]{2,}$/g

Número: (12) 12345-6789
RegEx Número:/[(][0-9]{2}[)][ ][0-9]{4,5}[-][0-9]{4}/g

CPF: 123.456.789-12
RegEx CPF: /[0-9]{3}[.][0-9]{3}[.][0-9]{3}[-][0-9]{2}/g
