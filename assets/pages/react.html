Introdução ao React

O que é o React

O React é uma Framework/biblioteca JavaScript para Frontend.

Utilizado para construir SPAs (Single Page Application)

Baseado em Componentes.

Utiliza JSX para renderizar HTML (JavaScript and XML)

Aplica o Virtual DOM para realizar alterações no DOM

Podemos adicionar a um projeto ou criar um projeto com ele

Criação Projeto React

Antigamente utilizava-se a ferramente 'Create React App', porém essa ferramente criava muitos documentos que não iam ser utilizados e dificultava a manutenção no código, então agora é utiliado 'Vite' para criar.

Criação do Projeto:

  • Criar a Pasta do Projeto
  • Abrir ela no cmd
  • Digitar npm create vite@latest
  • Confirmar com y (se for a primeira vez)
  • Definir o nome do projeto
  • Escolher o Framework
  • Escolher Variante (melhor pegar com SWC)
  • Ir para a pasta Criada (através do cmd)
  • Digitar npm install
  • Digitar npm run dev

React Basics

JSX -> é como HTML só que dentro do JavaScript

{ } -> Chaves podem ser usados para interpolar variáveis no conteúdo HTML

Os componentes são criados com funções que retornam um HTML. Toda função que é um componente inicia-se com letra maiúscula. Escreve-se o HTML dentro de () depois do return.

Toda vez que se quer que o react interprete algo como sendo código javascript basta colocar entre chaves { }

Um documento JSX tem que estar iteiro envolto à alguma tag. Ou seja, uma única tag pai.

O React reserva a palavra clas para outra coisa. Então quando queremos colocar uma class em uma tag html utilizamos 'className'

Components

Permite dividir aplicação em partes.

Renderizam JSX

Criar uma pasta e um arquivo para cada Componente.

Precisamos importar o componente onde quisermos utilizar.

Ficam normalmente em uma pasta chamada components.

Ao criar um novo componente, a função normalmente tem o mesmo nome do arquivo.

Para utilizar um componente, devemos importálo e dentro do return da função esrevemos o nome do componente entre uma tag < e />

Os componentes devem ser reutilizáveis.

Props

Props são valores passados para os componentes.

Podemos deixá-los dinâmicos.

É passado como um atributo na chamada do componente.

É preciso ser resgatado dentro de uma propriedade/argumento.

Props ex:
Component: 

    function SayMyName(props){
        return (
            <>
                <p>Hello {props.name}, how are you?</p>
            </>
        )
    }
App.jsx: 

    function App(){
        return (
            <>
                <SayMyName name="Lucas"/>
            </>
        )
    }

Podemos desestruturar o props com o nome das variáveis que queremos entre chaves { }

Prop Types

PropTypes é um pacote do react, para validar props e variáves. Como o faz o Typescript

Instalação

  • No terminal: npm i prop-types
  • No documento: import PropTypes from 'prop-types'
O pacote PropTypes serve para declarar propriedades das props. Se declara essas propriedades fora da função do componente

As propriedades são declaradas na forma de objeto javascript

PropTypes de Validação

ex: 
    || O nome do componente é Item
    Item.propTypes = {
        marca: PropTypes.string,
        ano_lancamento: PropTypes.number.isRequired
    }

Esta forma de validação não causará erro na execução do código, porém mostrará erro ao no console. Isso serve para que outros desenvolvedores leiam o seu código.

PropTypes de Valor Default


    Item.defaultProps = {
        marca: 'Faltou a Marca',
        ano_lancamento: 0
    }

    const [variavel, setVariavel] = useState(valor_da_variavel)
    setVariavel(alteração que acontecerá)

State Lift

  • State Lift é um técnica utilizada para compartilhar o state.
  • É normal vários componentes dependerem do mesmo estado;
  • Então precisamos elevar o nível do mesmo a um componente pai;
  • Então centralizamos o state no pai, e definimos quem usa e quem define (setState)
  • Serve para usar a mesma var do useState em vários componentes. Isso se faz criando no componente pai e se passa atrvés de props e parâmetro da função filha.

React Router

Sobre

React Router é um pacote externo ao react que serve para criar diferentes páginas com o react.

INSTALAÇÃO REACT ROUTER

  • No Terminal Digitar: npm install react-router-dom

CONFIGURAÇÃO REACT ROUTER

  • Dentro do SRC, criar a pasta Routes, que terá todas as notas páginas no formato de Componente
  • No main.jsx colocaremos os seguintes comandos:
    • import { createBrowserRouter, routerProvider } from "react-router-dom";
    • import __NOSSA PÁGINA__ from "__CAMINHO DA NOSSA PÁGINA__"
    • const router = createBrowserRouter([]); || Esse array vai receber objetos com as propriedades path, que receberá o caminho, e a propriedade element que receberá a tag do componente. A página Home recebe "path: "/" ", as outras recebem o nome da página.
    • Apagar o <App /> e colocar: <RouterProvider router={router} /> || Aquele router é a variável que foi criada anteriormente.
ex:

REAPROVEITAMENTO DE ESTRUTURA

  • É melhor ao invés de colocar as páginas assim, colocar um objeto só que é a página App.jsx, com o path: "/", element: <App /> e children: [__Colocar aqui as páginas__].
  • Então no App.jsx colocar: import { outlet } from "react-router-dom" e chamar a tag <Outlet /> onde queremos que o componente da página seja renderizado.
  • Assim podemos fazer com que uma página seja um componente renderizado dentro de uma página principal.

Use Effect

Visão Geral:

  • Executa uma função toda vez que algo acontecer
  • Recebe 2 parâmetros. 1 -> A função que será executada. 2 -> Um array que contenha as variáveis que ao atualizar, ativarão o useEffect()
  • Caso não seja passado o segudo parâmetro, ele executa toda vez que qualquer um dos componentes renderizarem.

Use Ref

Visão Geral:

  • A vantagem do useRef() é que quando atualiza seu valor, ele não atualiza re-renderiza o componente.
  • Caso um useEffect tenha dentro de si um useState e não tenha alguma variável de controle, ele cai num loop infinito. useRef() resolve esse problmea
  • Declar ref: const numberRef = useRef(0)
  • Recebe valor inicial dentro dos parenteses
  • useRef na verdade é um objeto que possui a propriedade current.
  • Para acessaro valor temos que usar o nome da variável.current
  • Isso faz com que o número altere na tela porem não re-renderize a página.
